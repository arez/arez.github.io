001package arez;
002
003/**
004 * Enum to control scheduling priority of observers/reactions.
005 * Observers with higher priorities will react first. If observers have equal priorities then observers
006 * scheduled first will react first. Observers must not depend upon ComputedValue instances with
007 * a lower priority otherwise priority is ignored.
008 *
009 * <p>A user should be very careful when specifying a {@link #HIGHEST} or {@link #HIGH} priority as it
010 * is possible that the the reaction will be scheduled part way through the process of disposing and/or
011 * unlinking one-or-more components. In many cases this may mean invoking
012 * <code>Disposable.isDisposed(component)</code> before accessing arez components.</p>
013 */
014public enum Priority
015{
016  /**
017   * Highest priority.
018   * This priority should only be used by reactions responsible for disposing or releasing reactive components.
019   * This priority is used when executing the reaction is likely to reduce the load of reactions that may react
020   * within the same round. Any reaction scheduled with this priority must guard against accessing partially or
021   * wholly disposed reactive elements.
022   */
023  HIGHEST,
024  /**
025   * High priority.
026   * This priority should be used when the reaction will dispose other reactive elements (and thus they
027   * need not be scheduled) or if the reaction will trigger many downstream reactions.
028   */
029  HIGH,
030  /**
031   * Normal priority if not otherwise specified.
032   */
033  NORMAL,
034  /**
035   * Lowest priority.
036   */
037  LOW
038}
